# 模型价格

模型价格页面维护 CPAMP 成本估算所需的价格表。它只影响面板里的成本展示，不会改变提供商账单，也不会改变模型路由。

如果你要看成本花在哪里，请去 [用量分析](./usage-analytics.md)。

## 什么时候需要改价格

- 用量分析里某个模型成本为空或明显偏低。
- 客户端使用了自定义模型名、别名或内部模型名。
- 上游价格发生变化。
- 你希望统一使用 LiteLLM 或 OpenRouter 的公开价格作为估算基准。

价格名必须和请求监控中看到的模型名匹配。模型路由别名、提供商实际模型名和价格表模型名不一致时，要按请求事件里的名称补价格。

## 常用操作

- 搜索模型价格。
- 新增或编辑模型价格。
- 删除不再使用的价格项。
- 从外部价格源主动同步。
- 保存后回到用量分析确认成本是否恢复。

同步外部价格只在你主动触发时发生。触发前先确认服务器网络和代理策略，避免误以为 CPAMP 会后台自动联网。

## 字段理解

不同模型的价格结构可能不同，常见字段包括输入 Token、输出 Token、缓存读写、推理 Token 或请求级费用。填写时遵循提供商公开价格说明。

如果某个字段没有对应价格，可以留空或按实际计费方式处理；不要为了让成本看起来完整而填入不确定价格。

## 验证方式

1. 在 [请求监控](./monitoring.md) 找到一条使用目标模型的请求。
2. 记录请求里的模型名。
3. 在模型价格页确认同名价格存在。
4. 回到 [用量分析](./usage-analytics.md)，刷新并查看该模型的预估成本。

## 准确性边界

- 提供商账单永远以提供商为准。
- CPAMP 只能根据已采集请求和当前价格表估算。
- 缺少 Token 字段、模型名被改写或价格表过期都会影响估算。
- 多币种、阶梯价、包月额度和赠送额度不会总是能被单一价格表准确表达。
