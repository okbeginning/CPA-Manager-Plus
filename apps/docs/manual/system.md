# 系统信息

系统信息页面用于确认当前面板连接到什么运行时、可用哪些模型，以及排障时应提供哪些基础信息。

日志内容请看 [日志查看](./logs.md)。请求、成本和账号状态请分别看 [请求监控](./monitoring.md)、[用量分析](./usage-analytics.md) 和 [认证文件](./auth-files.md)。

## 主要内容

- **快速链接**：打开主仓库、Web UI 仓库和文档入口。
- **模型列表**：从当前连接读取可用模型，并按来源分组展示。
- **登录存储清理**：清除浏览器里保存的登录连接信息。
- **连接状态**：用于判断当前是否已经连接到可用 CPA 或 Manager Server。

实际显示内容会随运行模式变化。Full Docker、CPA Panel、原生包和演示模式下可用功能不同。

## 模型列表怎么用

模型列表适合回答两个问题：

1. 当前运行时能看到哪些模型。
2. 模型是来自哪个提供商或配置。

如果客户端请求某个模型失败，先在这里确认模型是否可见；如果不可见，再去 [AI 提供商](./ai-providers.md) 检查配置和模型规则。

模型可见不代表请求一定成功。认证、配额、上游状态和路由规则仍然可能导致请求失败。

## 清除登录存储

清除登录存储会删除浏览器里保存的面板连接信息，适合这些情况：

- 切换到另一个 CPAMP 或 CPA 地址。
- 管理员密钥变更后，浏览器一直使用旧连接。
- 本地开发或演示环境之间来回切换。

这不会删除服务器上的配置、认证文件、SQLite 数据或 CPA 配置。它只影响当前浏览器。

## 排障时建议提供

反馈问题前，先在系统信息页确认并记录：

- CPAMP 版本和运行模式。
- CPA 版本。
- 当前访问方式：Full Docker、CPA Panel、原生包或开发环境。
- 模型列表是否能正常加载。
- 是否刚清理过登录存储或更换过地址。

再结合 [日志查看](./logs.md) 和 [请求监控](./monitoring.md) 提供同一时间段的脱敏证据。
