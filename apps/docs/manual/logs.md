# 日志查看

日志查看页面用于排查运行时问题。它适合在请求监控摘要不够、OAuth 回调异常、插件资源空白或配置热重载失败时使用。

系统版本、运行模式和模型列表请看 [系统信息](./system.md)。

## 适合看日志的场景

- 提供商请求失败，但请求监控摘要不足以判断原因。
- OAuth 登录或回调失败。
- 插件加载、插件配置或插件页面资源异常。
- 配置保存后没有生效。
- 反向代理路径命中了错误后端。
- CPA 或 Manager Server 连接异常。

## 页面操作

- 刷新日志内容。
- 屏蔽管理面板自身产生的日志，聚焦模型请求或运行时错误。
- 切换原始日志，便于复制多行内容。
- 打开错误请求日志列表，下载对应时间点的错误日志文件。

如果当前开启了请求日志，CPA 的独立错误请求日志列表可能为空。这是运行时行为，不代表没有失败请求。此时优先使用 [请求监控](./monitoring.md) 查看失败摘要。

## 排障时怎么用

1. 先在请求监控或页面错误中确定大概时间点。
2. 回到日志页，刷新并定位同一时间段。
3. 复制脱敏后的错误行、状态码和上游错误摘要。
4. 如果日志指向认证或配额，继续到 [认证文件](./auth-files.md) 或 [配额管理](./quota.md)。
5. 如果日志指向插件，继续到 [插件管理](./plugins.md) 检查状态和资源路径。

## 不要泄露这些内容

日志中可能出现路径、请求头或上游摘要。反馈问题前请删除：

- CPA Management Key
- CPAMP Admin Key
- 提供商 API Key
- OAuth Token
- 完整认证文件内容
- 未脱敏的 Authorization Header
