# 配置中心

配置中心用于调整 CPAMP 和 CPA 的运行配置。它更像“运行时开关和连接设置”，不要把这里和 [AI 提供商](./ai-providers.md) 混在一起：配置中心决定系统怎么运行，AI 提供商决定模型请求转发到哪里。

## 页面入口

配置中心通常包含三类编辑方式：

- **可视化配置**：适合日常调整。字段分组展示，保存前可以检查变更。
- **源文件配置**：适合熟悉 `config.yaml` 的用户。改动更直接，但也更容易写错格式。
- **Manager Server 配置**：只在 Manager Server 托管面板时可用，用于绑定 CPA、启停采集器和保存 CPAMP 自己的运行设置。

如果某个 Tab 不可用，先确认当前是 Full Docker、CPA Panel 还是原生包模式。CPA Panel 模式下，一些 Manager Server 能力不会出现。

## Manager Server 配置

这里最容易影响请求监控和用量分析，保存前要看清楚：

- **CPA 地址**：Manager Server 访问 CPA 的地址。容器部署时通常是 Docker 网络里的服务名，不一定是浏览器访问地址。
- **CPA Management Key**：只用于 CPAMP 调用 CPA 管理接口，不是客户端请求模型时使用的 API Key。
- **请求监控**：开启后会启用 CPA 用量统计，并启动 CPAMP 采集器。
- **采集模式**：自动、RESP 或 HTTP。RESP 需要直连 CPA API 端口，不能走普通 HTTP 反向代理。
- **轮询间隔、批量大小、查询上限**：影响采集延迟和单次读取量。不要把轮询间隔设置得比 CPA 队列保留时间还长。
- **配置来源**：环境变量、SQLite 或未保存。环境变量优先级最高，面板保存不能覆盖环境变量。

## 可视化配置与源文件配置

日常建议优先用可视化配置，因为它会按功能分组并减少 YAML 格式错误。源文件配置适合这些场景：

- 需要一次性调整多段配置。
- 需要复制已有 `config.yaml`。
- 需要检查可视化配置没有覆盖的字段。
- 需要和部署环境中的配置做差异比对。

保存源文件前，先确认缩进、数组和字符串引号正确。保存后如果页面没有刷新，检查 CPA 是否支持热重载，或是否需要重启运行时。

## 保存后怎么验证

1. 回到仪表盘确认连接和采集器状态。
2. 进入 [AI 提供商](./ai-providers.md) 确认提供商配置仍然存在。
3. 发一条低成本请求。
4. 到 [请求监控](./monitoring.md) 看是否出现请求事件。
5. 如果改动涉及用量统计，到 [用量分析](./usage-analytics.md) 检查统计是否更新。

## 常见误区

- CPAMP 管理员密钥、CPA Management Key、客户端 API Key 和提供商 API Key 是四类不同密钥。
- 关闭 CPAMP 请求监控采集器，不等于清空 CPA 用量队列。
- 模型价格只影响 CPAMP 成本估算，不会改变提供商实际计费。
- 环境变量来源的配置要改部署环境并重启，不能只在面板里保存。
