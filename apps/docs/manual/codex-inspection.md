# Codex 账号巡检

Codex 账号巡检用来判断 Codex 账号为什么不能稳定服务请求。它比请求监控更偏账号状态：配额、认证、工作区、重置时间和可恢复动作。

如果你只是想看某条请求失败原因，先去 [请求监控](./monitoring.md)。如果已经确认问题集中在 Codex 账号，再进入本页。

## 检查内容

- 账号计划、配额窗口、重置时间和剩余额度。
- OAuth Token 是否过期或失效。
- 工作区状态是否影响请求。
- 账号是否触达 `usage_limit_reached`。
- 是否需要重新授权、恢复或禁用。

不同账号返回的信息可能不完整。缺失字段会按未知处理，不会被当成健康或异常。

## 本地与服务端巡检

页面提供两种方式：

- **本机巡检**：在当前浏览器会话发起，适合临时诊断和少量账号检查。
- **服务端巡检**：提交给 Manager Server，适合后台运行、定时巡检、历史记录和统一执行建议动作。

启用服务端巡检前先确认：

1. CPA URL 和 CPA Management Key 正确。
2. 认证文件中账号信息完整。
3. `auth_index` 稳定。
4. 账号处理策略符合预期，避免自动禁用误伤。

## 巡检结果怎么看

结果会按账号给出建议动作：

- **保留**：没有发现需要处理的问题。
- **重新登录**：OAuth 或认证状态失效，进入 [OAuth 登录](./oauth.md) 重新授权。
- **禁用**：账号暂时不适合继续服务请求，例如额度超过阈值或状态异常。
- **启用**：账号已经恢复，可以重新启用。
- **删除**：账号明确失效且确认不再使用时才执行。

不要只看建议动作，要同时看原因、证据和最近请求表现。对于有业务影响的账号，先禁用观察，不要直接删除。

## 服务端定时巡检

服务端巡检可以按固定间隔或每天指定时间点执行。保存配置后，通常会在下一个 worker 轮询周期内生效。

定时巡检适合：

- 大量 Codex 账号的日常健康检查。
- 自动恢复已过冷却期的账号。
- 生成可追溯的巡检历史和日志。

如果启用了自动动作，先从保守模式开始，只自动启用明确恢复的账号；删除和禁用应在观察一段时间后再考虑。

## 自动动作边界

自动化只处理明确可恢复的状态，例如配额冷却到期恢复。手动禁用的账号不会被自动恢复覆盖。

如果巡检结果和实际请求表现不一致，先查看请求监控的失败摘要，再检查认证文件和 OAuth 状态。
